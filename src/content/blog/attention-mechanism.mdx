## Introduction

Attention 메커니즘은 현대 딥러닝의 핵심 구성 요소입니다. 이 글에서는 Scaled Dot-Product Attention부터 Multi-Head Attention까지 수식과 코드로 정리합니다.

## Scaled Dot-Product Attention

Query($Q$), Key($K$), Value($V$)가 주어졌을 때, attention은 다음과 같이 계산됩니다:

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

여기서 $d_k$는 key의 차원입니다. $\sqrt{d_k}$로 나누는 이유는 dot product의 크기가 차원에 비례하여 커지는 것을 방지하기 위함입니다.

### PyTorch 구현

```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Args:
        Q: (batch, heads, seq_len, d_k)
        K: (batch, heads, seq_len, d_k)
        V: (batch, heads, seq_len, d_v)
        mask: optional attention mask
    """
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)

    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))

    attn_weights = F.softmax(scores, dim=-1)
    return torch.matmul(attn_weights, V), attn_weights
```

## Multi-Head Attention

단일 attention 대신 여러 "head"를 사용하면, 모델이 서로 다른 representation subspace에서 정보를 학습할 수 있습니다:

$$
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h)W^O
$$

각 head는:

$$
\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)
$$

### 구현

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0
        self.d_k = d_model // n_heads
        self.n_heads = n_heads

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, Q, K, V, mask=None):
        batch_size = Q.size(0)

        # Linear projections + reshape to (batch, heads, seq, d_k)
        Q = self.W_q(Q).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        K = self.W_k(K).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        V = self.W_v(V).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)

        # Attention
        out, attn_weights = scaled_dot_product_attention(Q, K, V, mask)

        # Concat + output projection
        out = out.transpose(1, 2).contiguous().view(batch_size, -1, self.n_heads * self.d_k)
        return self.W_o(out)
```

## Key Takeaways

- Attention은 **query와 key의 유사도**를 기반으로 value를 가중합하는 메커니즘
- Scaling factor $\frac{1}{\sqrt{d_k}}$가 학습 안정성에 중요
- Multi-head를 통해 **다양한 관점**에서 정보를 포착
- Self-attention은 $Q = K = V$인 특수한 경우
