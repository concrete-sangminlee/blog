연구는 컴퓨터 앞에서 하는 일이 많다. 마우스로 메뉴를 클릭하는 대신 단축키를 쓰면 시간이 꽤 절약된다. 내가 자주 쓰는 단축키들을 정리해본다.

## 공통

- **Cmd/Ctrl + C, V, X**: 복사, 붙여넣기, 잘라내기. 당연하지만 필수.
- **Cmd/Ctrl + Z, Shift+Z**: 실행 취소, 재실행. 실수를 되돌리는 마법.
- **Cmd/Ctrl + F**: 검색. 모든 프로그램에서 쓸 수 있다.
- **Cmd/Ctrl + A**: 전체 선택.
- **Cmd/Ctrl + S**: 저장. 자주 눌러야 한다.

## 브라우저

- **Cmd/Ctrl + T**: 새 탭. 하루에도 수십 번 쓴다.
- **Cmd/Ctrl + W**: 탭 닫기. 더 자주 쓴다.
- **Cmd/Ctrl + Shift + T**: 닫은 탭 다시 열기. 실수로 닫은 탭을 복구.
- **Cmd/Ctrl + L**: 주소창으로 커서 이동.
- **Cmd/Ctrl + 숫자**: N번째 탭으로 이동.

## 텍스트 에디터

- **Cmd/Ctrl + D**: 단어 선택 / 다음 동일 단어 선택 (VS Code)
- **Cmd/Ctrl + /**: 줄 주석 처리
- **Cmd/Ctrl + Shift + K**: 줄 삭제
- **Alt + Up/Down**: 줄 위아래로 이동
- **Cmd/Ctrl + Shift + L**: 모든 동일 단어 한 번에 선택

이것들만 써도 코딩 속도가 확연히 빨라진다.

## 터미널

- **Ctrl + R**: 이전 명령어 검색
- **Ctrl + A / E**: 줄 맨 앞 / 맨 뒤로 이동
- **Ctrl + L**: 화면 정리 (clear)
- **Ctrl + C**: 현재 프로세스 중단
- **Tab**: 자동 완성. 파일명, 명령어 모두.

## Git

- **gs**: git status (alias)
- **gd**: git diff
- **gc**: git commit
- **gp**: git push
- **gl**: git log

이건 단축키라기보다 shell alias다. `.bashrc`나 `.zshrc`에 설정해두면 git 명령이 훨씬 빨라진다.

## macOS 특유의 것

- **Cmd + Space**: Spotlight 검색. 앱 실행, 파일 찾기, 계산기 대용.
- **Cmd + Tab**: 앱 전환.
- **Cmd + Shift + 3/4**: 스크린샷.
- **Cmd + Q**: 앱 종료. 창 닫기(W)와 다르다.

## 윈도우 특유의 것

- **Win + D**: 바탕화면 보기
- **Win + E**: 파일 탐색기
- **Alt + Tab**: 앱 전환
- **Win + 화살표**: 창 배치 (왼쪽/오른쪽 반반)

## 배우는 순서

단축키를 한 번에 다 외우려 하면 포기하게 된다. 하나씩, 필요할 때마다 익히는 게 좋다.

"아, 이 동작을 자주 하네" 하는 게 있으면, 그 단축키부터 찾아보자. 한 번 익히면 평생 쓴다.

## 결론

단축키는 소소한 시간을 아껴준다. 하나당 몇 초지만, 하루에 수백 번 쓰면 합이 크다. 일 년이면 몇 시간, 몇 년이면 며칠을 아낄 수 있다.

무엇보다, 손이 키보드를 떠나지 않는 것의 쾌감이 있다.
