지금 내 브라우저에 탭이 몇 개 열려있는지 세어봤다. 27개다.

탭의 구성을 보면 내가 오늘 뭘 했는지 (또는 뭘 안 했는지)를 알 수 있다.

## 논문 관련 (8개)

- Google Scholar 검색 결과
- PDF 논문 3개 (다 읽지는 않았다)
- ScienceDirect 논문 페이지 2개
- Connected Papers 그래프 1개
- 인용 형식 변환 사이트

이 중에 실제로 꼼꼼히 읽은 건 1개. 나머지는 "나중에 읽어야지" 상태.

## 코딩 관련 (7개)

- GitHub 레포 2개
- Stack Overflow 3개 (같은 에러에 대한 다른 답변들)
- PyTorch 공식 문서 1개
- numpy reshape 관련 검색 결과 1개 (또)

Stack Overflow 탭이 3개인 건, 첫 번째 답변이 안 맞아서 다른 걸 찾은 흔적이다.

## 기타 (12개)

- 이메일 (Gmail)
- 캘린더
- 날씨
- 배달앱 (점심 뭐 먹을지 고민 중)
- 유튜브 (잠깐 쉬려고 열었는데 아직 안 닫음)
- 쇼핑몰 (키보드를 새로 살까 고민 중)
- 뉴스 기사 3개
- 이 블로그의 어드민 페이지
- 기타 잊어버린 것들

## 진단

논문 관련 탭보다 기타 탭이 더 많다. 이건 좋은 신호가 아니다.

하지만 탭을 닫으면 "나중에 다시 못 찾으면 어쩌지" 하는 불안감이 있다. 북마크를 하면 되는데, 그것도 귀찮다.

결국 탭은 계속 쌓이고, 메모리는 줄어들고, 팬은 돌아가고. 노트북 편지에서 사과해야 할 항목이 하나 더 늘었다.
