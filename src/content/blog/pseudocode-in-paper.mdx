논문에 알고리즘 설명. pseudocode. 실제 코드가 아닌, 코드 같은 설명.

## 왜 pseudocode

왜? 구체적 코드는 언어에 종속. pseudocode는 언어 무관.

누구나 이해.

## 형식

형식.

```
Algorithm 1: Name
Input: x
Output: y
1: for i = 1 to n do
2:     compute z
3: end for
4: return y
```

번호 매기기, 들여쓰기.

## LaTeX

LaTeX의 `algorithm` 패키지. 자동 포맷.

## 명확성

명확해야 한다. 독자가 알고리즘을 이해. 실행도 가능한 정도.

## 추상화

너무 구체적이면 코드. 너무 추상적이면 이해 어렵다. 중간.

## 영어 문장

영어 문장 섞어서. "For each data point, compute..." 자연어 사용 가능.

## 수학 기호

수학 기호 쓸 수 있다. $\sum$, $\in$, $\forall$.

## 간결

간결하게. 한 줄 한 줄 의미 있게. 불필요한 것 제거.

## 여러 알고리즘

여러 알고리즘. "Algorithm 1", "Algorithm 2." 번호 매기기.

## 본문에서 참조

본문에서 참조. "Algorithm 1에서 보듯이..."

## 설명

pseudocode 이후 설명. 주요 단계 설명. 독자 도움.

## 내가 처음

처음 pseudocode 썼을 때 어색. 형식 배우기.

지금은 익숙.

## 스타일

스타일. 저자마다 다르다. 일관성이 중요.

## 복잡한 알고리즘

복잡한 알고리즘. pseudocode가 여러 페이지. 나누기.

## 간단한 것

너무 간단하면 pseudocode 불필요. 본문에서 설명.

## 복잡도 분석

복잡도 분석도. "Algorithm 1의 복잡도는 O(n²)." 학술적.

## 결론

논문에 pseudocode 쓰기는 알고리즘을 명확히 전달하는 도구다. 독자와의 소통.

다음 논문에 알고리즘이 있으면, pseudocode로 표현하자. 명확성 향상.
