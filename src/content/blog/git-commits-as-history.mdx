연구 프로젝트의 git log를 보면, 그게 연구 일기 같다. 제대로 쓴 커밋 메시지는 언제 어떤 생각을 했는지 기록이 된다.

## 초반의 희망

```
init project
add data loader
implement baseline model
```

간결하고 체계적이다. 이때는 의욕이 넘쳤다.

## 중반의 혼돈

```
fix bug
fix another bug
fix the fix
why is this not working
WIP
```

점점 메시지가 짧아지고, 감정이 들어간다. "WIP"는 나중에 나도 뭔지 모른다.

## 새벽의 절망

```
i give up
come back tomorrow
```

타임스탬프를 보면 새벽 3시다. 이런 커밋은 나중에 봐도 웃프다.

## 깨달음의 순간

```
FINALLY working
IT WORKS
breakthrough!
```

대문자와 느낌표가 나타나면 뭔가 해결됐다는 신호다. 이 전후의 변경 사항을 보면 보통 사소한 버그 하나였다.

## 논문 마감

```
for paper submission
final version
final version v2
final version FINAL
really final
```

"final"이 여러 번 반복되는 건 논문 마감의 정석이다.

## 결론

커밋 메시지는 단순한 기록이 아니라 감정의 기록이기도 하다. 나중에 이걸 다시 보면 그때의 자신이 보인다.

참고로 커밋 메시지를 제대로 쓰는 건 좋은 습관이다. 미래의 내가 감사할 거다. 나처럼 "fix bug"만 쓰지 말고.
