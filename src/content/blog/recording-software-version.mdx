논문 작성 중. 방법론 섹션. "사용 소프트웨어: Python 3.9, NumPy 1.21, TensorFlow 2.8." 버전 꼼꼼히 기록.

## 왜 중요

왜 중요. 재현성. 다른 사람이 같은 환경에서 실행 가능하게.

## 버전 차이

버전 차이. 같은 코드인데 다른 결과. 기능 변경, 버그 수정.

## 기록

기록. 논문에 명시. 또는 깃허브 readme에.

## requirements.txt

requirements.txt. Python 프로젝트의 버전 기록 파일. 표준.

## 환경 파일

conda env.yml. 또는 Docker 이미지. 환경 통째로 보존.

## 모든 패키지

모든 패키지. 주요 것만 아니라. 작은 라이브러리도.

## 놓치기 쉬움

놓치기 쉽다. 매일 쓰는 것은 당연하게 여긴다. 기록 안 함.

## 나중에

나중에 곤란. "이 코드 어떻게 돌리지?" 환경 모른다.

## 재현 실패

재현 실패. 다른 사람이 결과 못 얻음. 문제.

## 버전 관리

버전 관리. Git. 코드의 변화 추적.

코드 버전 + 패키지 버전.

## 도커

Docker. 환경 통째로 이미지. 재현성 최고.

하지만 복잡.

## 간단한 방법

간단한 방법. 논문에 버전 쓰기. 그게 최소.

## 회고

옛 논문 볼 때. 버전 확인. "그땐 이런 버전 썼구나." 역사.

## 새 버전 호환

새 버전이 나오면 호환성 문제. 옛 코드 안 돌아갈 수도.

## 좋은 습관

좋은 습관. 프로젝트 시작할 때 환경 기록.

requirements.txt를 먼저 만들기.

## 자동화

자동화. `pip freeze > requirements.txt`. 한 명령어.

## 결론

논문 소프트웨어 버전 기록은 재현성의 기초다. 작은 기록이 큰 도움.

다음 프로젝트 시작할 때 버전 기록부터 하자. 미래의 자신에게.
