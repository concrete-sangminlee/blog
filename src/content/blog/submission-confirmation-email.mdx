논문 제출 버튼을 누른다. "Submit." 잠깐 후 이메일 알림.

"Your manuscript has been received."

이메일 하나로 안도.

## 즉각적

대부분 즉시 온다. 몇 초 내. 자동 시스템.

## 내용

내용. 제출 완료 확인. 논문 ID 번호. 저자들 목록.

공식 문서 같다.

## ID 번호

논문 ID 번호가 생긴다. 이제 이 논문은 시스템 안에 존재. 추적 가능.

"MS-2026-0417-1234."

## 공저자들에게

확인 메일이 공저자들에게도 간다. 모두가 동시에 알게 된다.

## 다시 한 번

다시 한 번 확인. 메일함에서 확인 메일을 열어본다. "정말 제출됐구나."

## 안 오면

가끔 안 온다. 스팸함 확인. 5분 후에도 없다. 걱정.

"실패했나."

## 재제출

가끔 시스템 오류. 다시 제출해야. 확인 메일이 두 개 오면 혼란.

## 저장

이 메일을 저장. 언젠가 필요할 수도. 전자 서류철에.

## 마음의 안도

확인 메일을 받으면 마음이 놓인다. 몇 달 또는 몇 년의 작업이 마침표.

## 축하

공저자들끼리 "축하합니다" 메시지. 확인 메일이 시작의 신호.

## 긴 대기

이제 리뷰 대기. 몇 달. 길다.

하지만 제출은 끝났다. 일단 잠깐 쉬자.

## 결론

논문 제출 후 확인 메일은 작은 이정표다. 안도의 순간.

다음 제출 후엔 확인 메일을 꼭 저장하자. 그 순간의 기록.
