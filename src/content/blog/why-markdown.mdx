처음 Markdown을 봤을 때 "이게 뭐야?"라는 반응이었다. `# 제목` `**굵게**` 같은 기호들. 워드 같은 위지윅 에디터가 더 편할 것 같았다.

그런데 한 번 쓰기 시작하니 안 쓸 수가 없게 됐다.

## Markdown이 뭔가

간단한 텍스트로 서식을 표현하는 방법이다. 별다른 도구 없이도 쓸 수 있다.

```
# 큰 제목
## 작은 제목

**굵게** *기울임*

- 항목 1
- 항목 2

[링크](https://example.com)
```

이게 다다. 특별한 학습이 필요 없을 정도로 간단하다.

## 처음의 거부감

처음에는 거부감이 있었다. "왜 굳이 기호를 외워? 워드에서 버튼만 누르면 되잖아."

하지만 써보니 다른 게 보였다.

## 장점

**가볍다**: 텍스트 파일이라 어떤 에디터에서든 열린다. 워드 파일처럼 무겁지 않다.

**버전 관리**: Git에 넣으면 변경 사항을 추적할 수 있다. 워드는 이게 어렵다.

**미래 호환**: 20년 후에도 텍스트 파일은 열린다. 워드 파일은 모를 수 있다.

**플랫폼 독립**: Mac, Windows, Linux 어디서든 같다. 모바일에서도 쓸 수 있다.

**키보드만으로**: 마우스 안 써도 된다. 손이 키보드를 떠나지 않는다.

## 어디에 쓰나

**노트 작성**: 회의 메모, 연구 일지, 아이디어 정리. 거의 모든 메모를 Markdown으로.

**README 파일**: GitHub에 코드를 올릴 때 README는 Markdown이 표준이다.

**블로그 글**: 이 블로그도 Markdown으로 쓰고 있다. 본문은 텍스트, 변환은 자동.

**문서 작성**: 가벼운 문서는 워드 대신 Markdown으로. 기술 문서에 특히 적합.

## 도구

Markdown을 지원하는 도구는 많다.

**Obsidian**: 가장 많이 쓰는 도구. Markdown 기반.
**Typora**: 위지윅처럼 보이지만 내부는 Markdown.
**VS Code**: 코드 에디터지만 Markdown 미리보기 가능.
**HackMD, StackEdit**: 온라인 Markdown 에디터.

## 한계

Markdown이 만능은 아니다.

**복잡한 레이아웃**: 표가 복잡해지거나, 다단 구성을 하려면 어렵다. 이런 건 워드나 LaTeX이 낫다.

**수식**: 기본 Markdown은 수식 지원이 없다. LaTeX 확장이 필요하다.

**디자인**: 문서를 예쁘게 꾸미는 데는 한계가 있다. 색상, 폰트 같은 디테일이 제한적.

가벼운 글에는 좋지만, 정식 보고서나 논문에는 다른 도구가 필요하다.

## 학습 곡선

처음 며칠은 기호를 외우는 게 어렵다. `**`, `__`, `>`, `-`, `*`, ` ``` ` 등.

그런데 1~2주만 쓰면 손이 자동으로 움직인다. 워드 단축키를 외우는 것보다 훨씬 쉽다.

## 추천

Markdown을 안 써본 사람에게 추천한다. 간단한 노트부터 시작해보자. 곧 Markdown 없이 메모하는 게 어색해질 거다.

특히 코딩하는 사람이라면 거의 필수다. README, 문서, 노트 모두 Markdown이다.

## 결론

Markdown은 단순한 기호 모음이지만, 글쓰기의 효율을 크게 바꿔준다.

처음에는 어색해도, 한 번 익숙해지면 워드로 못 돌아간다. 가벼움의 맛을 알게 된다.

오늘부터라도 메모 한 줄을 Markdown으로 써보자.
