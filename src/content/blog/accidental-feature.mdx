같은 소프트웨어를 몇 년 쓰다 보면, "이걸 다 안다"고 생각한다. 그러다 어느 날 우연히 새로운 기능을 발견한다. "이런 게 있었어?!"

## 우연의 발견

어떻게 발견했는지 기억도 안 난다. 단축키를 잘못 눌렀거나, 메뉴를 클릭했다가, 아니면 옆 사람이 쓰는 걸 보고.

"이게 뭐지?" 하고 시도해본다. 그리고 깨닫는다. "이거 진짜 유용하네."

## VS Code의 멀티 커서

처음 발견했을 때 충격이었다. Cmd+D를 누르면 같은 단어가 차례로 선택된다. 그리고 동시에 편집할 수 있다.

여러 곳을 한 번에 수정해야 할 때, 이 기능 없이 어떻게 살았나 싶다.

## Excel의 빠른 채우기

Excel에서 Ctrl+E를 누르면, 패턴을 인식해서 자동으로 채워준다. 이름에서 성만 추출하기, 이메일에서 도메인만 가져오기. VLOOKUP 없이도 가능한 작업.

이걸 알고 나서 데이터 정리 시간이 확 줄었다.

## Git의 stash

작업 중인 코드를 잠시 보관해야 할 때 쓴다. `git stash`로 임시 저장하고, 다른 브랜치로 옮겨가서 일하고, 다시 돌아와서 `git stash pop`으로 복원.

이걸 모를 때는 임시 커밋을 만들어서 일을 했다. 비효율적이었다.

## Mac의 Spotlight 계산기

Cmd+Space를 눌러서 Spotlight 검색창을 열고, 계산식을 입력하면 답이 나온다. "1234*5678"을 입력하면 결과가 즉시 표시된다.

계산기 앱을 안 열어도 된다.

## 브라우저의 탭 검색

크롬에서 Cmd+Shift+A (또는 Ctrl+Shift+A)를 누르면, 열려있는 모든 탭을 검색할 수 있다. 탭이 30개씩 열려있을 때 유용하다.

이전에는 탭들 사이를 스크롤하면서 찾았다. 시간 낭비였다.

## Python의 walrus operator

`:=` 라는 연산자다. Python 3.8에서 추가됐다. 변수 할당을 표현식 안에서 할 수 있다.

```python
if (n := len(data)) > 10:
    print(f"List is too long ({n} elements)")
```

이거 알고 나서 코드가 좀 더 간결해졌다.

## "왜 안 가르쳐줬을까"

이런 기능을 발견할 때마다 "왜 처음부터 안 가르쳐줬지?"라는 생각이 든다.

하지만 모든 기능을 한 번에 다 가르치는 건 불가능하다. 기본기부터 시작해서, 필요해지면 그때그때 새 기능을 배우는 게 자연스러운 과정이다.

## 새 기능을 찾는 법

**튜토리얼 다시 보기**: 익숙한 도구의 튜토리얼을 다시 보면 새로운 발견이 있다.

**옆 사람 관찰**: 같은 도구를 쓰는 동료가 어떻게 쓰는지 보면 배울 게 있다.

**문서 가끔 읽기**: 도구의 공식 문서를 주기적으로 훑어보면 새 기능 정보를 얻는다.

**Twitter/Reddit**: 개발자 커뮤니티에서 팁을 공유한다. 가끔 보면 유용하다.

## 결론

도구를 마스터하는 건 평생의 과정이다. 익숙해졌다고 생각해도, 새로 발견할 게 항상 있다.

새로운 기능을 발견하면 한동안 즐겁다. "이걸 일찍 알았으면 좋았을 텐데"라는 후회와 함께.

오늘도 어딘가에서 새 기능 하나를 발견할지 모른다.
