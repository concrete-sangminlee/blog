같은 소프트웨어를 몇 년 쓰다 보면 어느 순간 "이 정도면 다 안다"고 생각하게 된다. 그러다 아무렇지 않은 날 숨어 있던 기능 하나를 발견하고 놀란다. "이런 게 있었어?"

어떻게 발견했는지는 대체로 기억에 남지 않는다. 단축키를 잘못 눌렀거나, 메뉴를 괜히 열어봤거나, 옆 사람이 쓰는 모습을 곁눈질로 보다가. "이게 뭐지?" 하고 따라 해보면, 곧 이걸 왜 이제야 알았을까 싶은 문장이 자동으로 따라온다.

## VS Code의 멀티 커서

처음 발견했을 때 꽤 충격이었다. Cmd+D(Mac) 또는 Ctrl+D(Windows, Linux)를 누르면 같은 단어가 차례로 선택되고, 선택된 위치를 동시에 편집할 수 있다.

여러 곳을 한 번에 수정해야 할 때, 이 기능 없이 어떻게 살았나 싶다.

## Excel의 빠른 채우기

Excel에서 Ctrl+E를 누르면 패턴을 인식해서 자동으로 채워준다. 이름에서 성만 추출하거나, 이메일에서 도메인만 가져오는 식의 작업이 함수 없이도 가능하다.

함수를 외우지 않아도 되는 상황이 늘어난다.

## Git의 stash

작업 중인 코드를 잠시 보관해야 할 때 쓴다. `git stash`로 임시 저장하고, 다른 브랜치로 옮겨가서 일하고, 다시 돌아와서 `git stash pop`으로 복원.

이 기능을 모르면 "일단 커밋"으로 브랜치가 지저분해지기 쉽다. stash는 그 자잘한 커밋을 만들기 전 단계에서 쓰기 좋은 도구다.

## Mac의 Spotlight 계산기

Cmd+Space로 Spotlight 검색창을 열고 계산식을 입력하면 답이 바로 나온다. `1234*5678`처럼 입력하면 결과가 즉시 표시된다.

계산기 앱을 안 열어도 된다.

## 브라우저의 탭 검색

크롬에서 Cmd+Shift+A 또는 Ctrl+Shift+A를 누르면 열려 있는 모든 탭을 검색할 수 있다. 탭이 30개씩 열려 있을 때 특히 유용하다.

탭이 쌓여 있을 때 탭 줄을 눈으로 훑으며 찾는 것보다 훨씬 빠르다.

## Python의 walrus operator

`:=`라는 연산자다. Python 3.8에서 추가됐고, 변수 할당을 표현식 안에서 할 수 있게 해준다.

```python
if (n := len(data)) > 10:
    print(f"List is too long ({n} elements)")
```

조건문 안에서 값을 재사용해야 할 때 코드가 한 줄 짧아진다.

## "왜 안 가르쳐줬을까"

이런 기능을 발견할 때마다 "왜 처음부터 안 가르쳐줬지?"라는 생각이 든다.

하지만 기능 전체를 한 번에 다 가르치는 건 불가능하다. 기본기부터 시작해서, 필요해지면 그때그때 새 기능을 배우는 게 자연스러운 과정이다.

## 새 기능을 찾는 법

**튜토리얼 다시 보기**: 익숙한 도구의 튜토리얼을 다시 보면 새로운 발견이 있다.

**옆 사람 관찰**: 같은 도구를 쓰는 동료가 어떻게 쓰는지 보면 배울 게 있다.

**문서 가끔 읽기**: 도구의 공식 문서를 주기적으로 훑어보면 놓쳤던 기능을 발견할 수 있다.

**커뮤니티 살펴보기**: X/Twitter, Reddit, 개발자 블로그에는 실전 팁이 자주 올라온다.

## 계속 발견하는 사람

도구를 완전히 마스터하는 일은 현실적으로 어렵다. "이만하면 다 안다"는 감각은 보통 착각에 가까웠고, 3년 쓴 도구에서 새 기능을 발견하는 날이 여전히 찾아왔다.

그래서 새 기능을 발견했을 때의 감정은 단순한 기쁨이 아니었다. "왜 이걸 이제야 알았을까"라는 옅은 억울함이 늘 섞여 있었다. 다만 그 감정이 반복될수록, 도구에 대한 자기 지식이 미완성 상태라는 사실 자체는 편해졌다.
